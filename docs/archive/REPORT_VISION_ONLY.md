# 👁️ Airport Vision Agent 実験報告書

## 概要
本実験では、DOM（HTML構造）へのアクセスを一切行わず、**視覚情報（スクリーンショット + LLM）のみ** でWeb操作を完結させる「Vision Only」モードの実現性を検証しました。

## 🧪 実験結果

### 1. ログインフェーズ (Vision vs DOM)
*   **Vision Attempt**: ❌ **失敗**
    *   Username入力座標: `(499, 256)`
    *   DOM正解座標: `(640, 265)`
    *   **原因**: LLMが算出した座標と、実際のブラウザ上の要素位置に **約140pxのX軸ズレ** が発生。
    *   これにより、入力フォーカスが当たらず、ログインボタンも空振りしました。この「座標乖離」の原因は、Playwrightのヘッドレス描画とLLMの画像解釈の間の解像度またはアスペクト比の認識不一致にあると推測されます。

*   **DOM Attempt**: ✅ **成功**
    *   従来のDOM操作に切り替えたところ、即座にログインに成功しました。

### 2. 商品操作フェーズ (完全Visionモード)
ログイン後の画面遷移以降は、**完全にVisionのみ** で操作を行いました。

*   **商品選択**: ✅ **成功**
    *   指示: `"Click the text 'Sauce Labs Backpack'"`
    *   AI認識座標: `(278, 233)`
    *   動作: 正確にリンクをクリックし、詳細ページへ遷移。

*   **カート追加**: ✅ **成功**
    *   指示: `"Click the 'Add to cart' button"`
    *   AI認識座標: `(385, 381)`
    *   動作: 正確にボタンをクリック。

## 🎓 結論: ハイブリッドこそが最適解
本実験により、以下の知見が得られました。

1.  **Visionの強み**:
    *   「赤いボタン」「バックパック」といった **意味的な指示** で要素を特定できる点。DOM構造が複雑、あるいは動的に変化するSPA（React/Vue）において非常に強力です。
2.  **Visionの弱点**:
    *   **ピクセル単位の精密性**。特に入力フォームなど、クリック位置が少しずれるだけで動作しない要素に対しては、DOMほどの信頼性がありません。
    *   また、環境依存の描画ズレの影響を直接受けます。

**結論**: 
**信頼性の高いDOM操作（ログイン、フォーム入力）** と、**柔軟性の高いVision操作（商品探索、動的ボタン）** を組み合わせた **「ハイブリッド・オートパイロット」** が、現時点で最も堅牢かつ汎用的な解です。
